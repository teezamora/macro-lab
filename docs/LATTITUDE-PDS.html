1 Design Philosophy

LATTITUDE's visual identity is built around a dark, high-contrast aesthetic that feels premium and athletic — closer to a sports performance dashboard than a wellness app. The design conveys precision, focus, and seriousness without being harsh or intimidating.

Design Principles

PrincipleApplication
Dark firstDeep near-black backgrounds (#0a0b0c) reduce eye strain during long use and make macro numbers pop visually.
Data is the heroMacro numbers are large, bold, and colorful. Chrome and decoration are minimal. Nothing competes with the numbers.
Color carries meaningProtein is always green. Carbs are always yellow. Fat is always orange. Consistent across every chart, table, and card.
Monospace for dataAll numbers, codes, and labels use DM Mono. Proportional fonts for prose and UI text.
Micro-interactionsCards lift on hover. Pills animate active state. Inputs glow on focus. Nothing feels static.
Progressive disclosureSimple surface with depth available. The dashboard is clean; the builder reveals detail when needed.

4 Data Architecture

Profile Schema

Stored in both localStorage (key: lattitude_profile) and Firestore (users/{uid}/profile).

// Profile object structure
{
  units:      "imperial",  // "imperial" | "metric"
  weightKg:   83.9,        // always stored in kg internally
  heightCm:   180.3,       // always stored in cm internally
  age:        28,
  activity:   1.55,        // TDEE multiplier
  phaseAdj:   -400,        // kcal adjustment for prep phase
  phaseName:  "cut",       // "maintenance" | "bulk" | "cut" | "peak"
  bmr:        1950,
  tdee:       3022,
  targets: {
    calories: 2622,
    protein:  185,
    carbs:    230,
    fat:      65
  }
}

Recipe Schema

Defined in recipes.js as entries in the RECIPES array.

{
  file:        "cottage-cheese-pancakes.html",
  name:        "Cottage Cheese Pancakes",
  description: "High-protein batter · 7 ingredients · 45g each",
  emoji:       "🥞",
  thumbBg:     "linear-gradient(135deg, #1a1500 0%, #2d2400 50%, #1a1500 100%)",
  tags:        ["Breakfast", "High Protein", "Post-Workout"],
  servings:    17,
  perServing: {
    calories: 100,
    protein:  5.6,
    carbs:    8.9,
    fat:      4.6
  },
  macroSplit: {  // % of calories — must total 100
    protein: 22.6,
    carbs:   35.8,
    fat:     41.6
  }
}

Firestore Schema

// Collection structure
users/
  {uid}/
    profile: { /* profile object above */ }
    // future: recipes/{recipeId}, logs/{date}, workouts/{id}

Security Rules

rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /users/{userId}/{document=**} {
      allow read, write: if request.auth != null
        && request.auth.uid == userId;
    }
  }
}

5 User Flows

Flow 1 — First-time Setup

1
Land on Dashboard
Goals bar shows "Set Up Goals →" CTA. Recipe grid shows saved recipes.
2
Click "Set Up Goals"
Navigate to profile.html
3
Enter stats
Weight, height, age. Right sidebar shows live BMR/TDEE calculation as user types.
4
Select activity level and prep phase
Calorie target and macro targets update live. Protein/carbs/fat displayed in macro cards.
5
Sign in with Google + Save
Profile saves to localStorage and Firestore. Toast confirmation. Redirect to dashboard.
6
Dashboard shows goals
Goals bar now shows prep phase + daily protein/carbs/fat/calories.

Flow 2 — Building a Recipe

1
Click "+ Build Recipe"
Navigate to builder.html, Builder tab active.
2
Fill in Recipe Setup
Name, emoji, description, estimated servings, tags.
3
Add ingredients
For each ingredient: enter name, raw gram weight, and per-100g macro values from USDA. Press Enter or click Add.
4
Watch live dashboard
Sidebar macro dashboard updates after each ingredient. Donut ring and % bars update in real time.
5
Weigh finished batch
After cooking, enter the total cooked weight. 3 serving suggestions appear. Click "Best for goals" or choose manually.
6
Copy export snippet
Click "Copy Snippet" to get the recipes.js object. Paste into recipes.js and create matching HTML file.

Flow 3 — Reverse Engineering a Meal

1
Open builder.html → Reverse Engineer tab
2
Set targets
Enter desired protein, carbs, fat. Or click "Load my daily goals" to pull from saved profile (÷ 4 for per-meal).
3
Add first ingredient
Enter per-100g values. Solve hints appear: "To hit remaining 40g protein: use 157g → click Use →"
4
Click "Use →"
Gram field auto-fills. Click Add. Progress bars in sidebar update.
5
Add remaining ingredients
Repeat with carb and fat sources. Sidebar shows remaining budget for each macro.
6
Status badge reads "On Target"
All three macros within 2g of target. Meal plan is complete.

Flow 4 — Using on a Second Device

1
Open LATTITUDE on phone/new device
No profile locally. Goals bar shows "Set Up Goals" CTA.
2
Click "Sign in with Google"
Google popup opens. User signs in with same account.
3
Profile loads from Firestore
Goals bar immediately shows saved prep phase and macro targets. localStorage cache written for offline use.

Adding a New Recipe

  1. Build the recipe in builder.html or create recipes/your-recipe-name.html by duplicating an existing recipe page
  2. Copy the export snippet from the builder and add it to the RECIPES array in recipes.js
  3. Push to GitHub — the recipe appears on the dashboard automatically
Single source of truth recipes.js is the only file that drives the dashboard grid, filter pills, stats bar, and recipe count. The individual HTML files only handle the detail page. Never duplicate recipe metadata.
